Implementing Webhook Verification

Turning cryptographic theory into a bulletproof Next.js Route Handler.

In Lesson 11, we learned that exposing our local server via ngrok requires us to verify incoming webhooks using Svix to prevent forgery. Today, we implement this verification logic in a Next.js App Router endpoint.

The Implementation Steps

When Clerk sends a webhook, it includes three critical headers: svix-id, svix-timestamp, and svix-signature. To verify these in JavaScript, we use the official svix library.

1. The Route Handler Setup

In Next.js, we create a file at app/api/webhooks/clerk/route.ts. Because webhooks send raw JSON, we must extract the body and the headers from the incoming Request object.

import { Webhook } from 'svix'
import { headers } from 'next/headers'
import { WebhookEvent } from '@clerk/nextjs/server'

export async function POST(req: Request) {
  const WEBHOOK_SECRET = process.env.WEBHOOK_SECRET
  if (!WEBHOOK_SECRET) throw new Error('Missing WEBHOOK_SECRET')

  // Get the headers
  const headerPayload = await headers()
  const svix_id = headerPayload.get('svix-id')
  const svix_timestamp = headerPayload.get('svix-timestamp')
  const svix_signature = headerPayload.get('svix-signature')

  // If there are no Svix headers, error out
  if (!svix_id || !svix_timestamp || !svix_signature) {
    return new Response('Error: Missing svix headers', { status: 400 })
  }
  
  // Get the raw body
  const payload = await req.json()
  const body = JSON.stringify(payload)
  
  // ... verification comes next
}

2. Verifying the Payload

We instantiate a new Svix Webhook object using our secret, and call the verify() method. If the cryptographic signature does not match the payload and the secret, verify() will throw an error. Therefore, we must wrap it in a try/catch block.

  const wh = new Webhook(WEBHOOK_SECRET)
  let evt: WebhookEvent

  try {
    evt = wh.verify(body, {
      'svix-id': svix_id,
      'svix-timestamp': svix_timestamp,
      'svix-signature': svix_signature,
    }) as WebhookEvent
  } catch (err) {
    console.error('Error verifying webhook:', err)
    return new Response('Error occured', { status: 400 })
  }

  // If we reach here, the request is 100% authentic!
  console.log(`Received webhook: ${evt.type}`)
  return new Response('Webhook received', { status: 200 })

3. Handling the Event

Once verified, the evt.type property tells us what happened (e.g., user.created). We can then safely extract the user's data from evt.data and sync it with our PostgreSQL database, knowing with mathematical certainty that the event was actually dispatched by Clerk.
