Implementing SaaS Billing with Stripe

Turning your authenticated software into a revenue-generating business.

In Lesson 18, we mastered the theory of Webhooks. Now, we apply that theory to the most critical component of any B2B SaaS: Billing.

When you integrate Stripe (or Paddle, or LemonSqueezy) into a Next.js application, you are creating a distributed system. Stripe owns the "Financial Truth" (credit cards, subscription status, invoices). Your application owns the "Feature Truth" (what the user is allowed to click on). Webhooks act as the bridge.

1. The Database Architecture for Subscriptions

To gate premium features, your local database needs to know if a user (or organization) has paid. Since ProjectForge is a B2B app (Lesson 15), the subscription belongs to the Organization, not the individual user.

// schema.prisma

model Organization {
  id               String   @id // Clerk org_id
  name             String
  
  // Stripe specific fields
  stripeCustomerId String?  @unique
  stripePriceId    String?
  
  // The source of truth for the UI
  subscriptionStatus String @default("incomplete") // active, past_due, canceled
  subscriptionEnd    DateTime?
}

2. Initiating Checkout (Server Action)

When a user clicks "Upgrade to Pro", you do NOT collect their credit card details directly. Doing so triggers massive PCI compliance nightmares. Instead, you create a Stripe Checkout Session and redirect them to Stripe's hosted payment page.

In Next.js, this is best handled via a Server Action.

'use server';

import Stripe from 'stripe';
import { auth } from '@clerk/nextjs/server';
import { db } from '@/lib/db';
import { redirect } from 'next/navigation';

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);

export async function createCheckoutSession() {
  const { orgId } = auth();
  if (!orgId) throw new Error("Unauthorized");

  // 1. Lookup the Organization to see if they already have a Stripe Customer ID
  const org = await db.organization.findUnique({ where: { id: orgId } });

  // 2. Create the Checkout Session
  const session = await stripe.checkout.sessions.create({
    customer: org?.stripeCustomerId || undefined,
    payment_method_types: ['card'],
    mode: 'subscription',
    line_items: [
      {
        price: process.env.STRIPE_PRO_PRICE_ID, // ID from Stripe Dashboard
        quantity: 1,
      },
    ],
    // MAGIC TRICK: We pass our internal Org ID into Stripe's metadata.
    // Stripe will send this back to us in the webhook, so we know who paid!
    client_reference_id: orgId,
    
    success_url: `https://my-app.com/dashboard?success=true`,
    cancel_url: `https://my-app.com/pricing`,
  });

  // 3. Redirect the user to Stripe's hosted URL
  if (session.url) {
    redirect(session.url);
  }
}

The client_reference_id Trick

When Stripe eventually sends the webhook telling you the payment succeeded, Stripe only knows the credit card details. How do you map that payment back to the exact Organization in your database? By attaching your internal orgId to the client_reference_id field during session creation, Stripe guarantees it will send that exact string back to you in the webhook payload.

3. The Webhook: Syncing Subscription State

Once the user pays, they are redirected to your success_url. But wait! Just because they arrived at the success page does NOT mean they are a Pro user yet. They are a Pro user only when the Webhook fires and updates the database.

Here is the Next.js Route Handler for the Stripe Webhook, combining Lesson 18's signature verification with database logic.

// app/api/webhooks/stripe/route.ts
import Stripe from 'stripe';
import { db } from '@/lib/db';

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);

export async function POST(req: Request) {
  const body = await req.text();
  const signature = req.headers.get('stripe-signature') as string;

  let event: Stripe.Event;

  try {
    event = stripe.webhooks.constructEvent(body, signature, process.env.STRIPE_WEBHOOK_SECRET!);
  } catch (error) {
    return new Response('Webhook signature verification failed.', { status: 400 });
  }

  // Handle the event
  try {
    switch (event.type) {
      case 'checkout.session.completed': {
        const session = event.data.object as Stripe.Checkout.Session;
        
        // Retrieve the magic ID we passed earlier
        const orgId = session.client_reference_id;

        if (orgId) {
          // Update the local database to unlock Pro features
          await db.organization.update({
            where: { id: orgId },
            data: {
              stripeCustomerId: session.customer as string,
              subscriptionStatus: 'active',
            },
          });
        }
        break;
      }
      
      case 'customer.subscription.deleted': {
        const subscription = event.data.object as Stripe.Subscription;
        
        // When canceled, Stripe gives us the Customer ID, not the Org ID.
        // So we look up the Org by Customer ID.
        await db.organization.update({
          where: { stripeCustomerId: subscription.customer as string },
          data: { subscriptionStatus: 'canceled' },
        });
        break;
      }
    }

    return new Response('Webhook Processed', { status: 200 });

  } catch (err) {
    console.error('Error updating database:', err);
    // CRITICAL: Return 500 so Stripe retries the webhook later!
    return new Response('Database Error', { status: 500 });
  }
}

4. Gating Features in the UI

With our local database perfectly synced to Stripe's financial reality, enforcing billing limits becomes a simple database lookup in your Server Components.

// app/dashboard/ai-generator/page.tsx
import { auth } from '@clerk/nextjs/server';
import { db } from '@/lib/db';
import { redirect } from 'next/navigation';

export default async function AIGeneratorPage() {
  const { orgId } = auth();
  if (!orgId) redirect('/sign-in');

  // Check the local source of truth
  const org = await db.organization.findUnique({ where: { id: orgId } });

  if (org?.subscriptionStatus !== 'active') {
    return (
      <div>
        <h2>Pro Feature</h2>
        <p>This feature requires an active subscription.</p>
        {/* Render an upsell component here */}
      </div>
    );
  }

  return <AIToolComponent />;
}
